    Hmm, this is how the algorithm goes for SBST...

    A, Y, U, S, P, L, J, G, D, B, Z, X, T, R, M, K, H, E, C

    From this list, the following letters are out: F, I, N, O, Q, V, W

    I don't know why some letters from the algorithm are left out, but I know why I and O are left out. This is because I resembles like '1', and O resembles '0'. As a result, both letters are omitted.

    Also, if you observe the algorithm through one or two sets, you might find out that some letters are left out from different sets of regos (e.g. 9810X, 9811T, 9812R, 9813M, 9814K, and 9828X, 9829T, 9830M, 9831K) The second set has left out the letter 'R'.

    Alright, my own algorithm for deriving the check digit is here. Before you read it, this is an important DISCLAIMER:

    A. I did NOT glean the info from espionage or any illegal way. I derived the algorithm through continued observations of many license plates (including private car license plates) of vehicles registered in Singapore over long periods, plus a good pattern sequencing intuition.

    B. Anyone with a good observation skills can possibly similarly derive this method of determining vehicle check digits.

    C. Since the method I describe here is rather long and tedious, this is probably NOT the official method or formula for deriving check digits. Although the method described here is 100% accurate, the official method is probably more direct and terse, possibly expressed in a mathematical formula.


    Okay, on to my algorithm:

    ========================
    The rule applies to ALL types of vehicle license plates in Singapore EXCEPT the following series:

    1. TP-XXXX
    2. ROV-XXXX
    3. MID-XXXXX
    4. S-XXXX-CD
    5. XXXX-S

    Rules for deriving check digits:

    1. The alphabets used for the check digits of vehicles in singapore are:
    ABCDEGHJKLMPRSTUXYZ

    2. The starting check digit of each series is arbitrary. e.g. TIB series starts with E, SBS series starts with Z.

    3. The ordering of the digits goes backwards and skips two digits between each consecutive number. Hence in the TIB series, one complete ordering cycle will be as follows:

    ECAYUSPLJGDBZXTRMKH - ECAY.....

    4. The derivation of each number's check digit from the next follows these rules, each with a higher priority over the other:

    a. Between every 999th to the next '000th, the same digit is used.
    e.g. since TIB999 is B, TIB1000 is also B

    b. Between every 99th to the next '00th, skip to the next 8th digit in the cycled ordering
    e.g. since TIB199 is T, TIB200 is Y

    c. Between every 9th to the next '0th, skip 2 digits in the cycled ordering
    e.g. since TIB29 is B, TIB30 is X

    d. Between every other type of consecutive number, skip to the next digit in the cycled ordering
    e.g. since TIB278 is Z, TIB 279 is X
    ========================

    Hence by going through the derivation rules, certain short cuts can be derived:

    a. From every X number to the (100 + X), skip 2 digits in the cycled ordering
    e.g. TIB1 is E, hence TIB101 is A

    b. From every X number to the (1000 + X), skip 12 digits in the cycled ordering
    e.g. TIB1 is E, hence TIB1001 is Z

    There are also other notable interesting observations:

    1. The SBS series is always 1000 ahead of the TIB series.
    e.g. SBS100 is X, TIB1100 is also X
    e.g. SBS41 is H, TIB1041 is also H
    Hence TIB(1000+X) always have the same check digit as SBS(X)

    2. Since the ordering is fixed, any series with the same check digit for the starting number WILL HAVE THE SAME CHECK DIGITS all the way for all the numbers 0-9999.
    e.g. the TIB, GP, YD, SBM, SCF, SCY, SH series are all TOTALLY IDENTICAL.

    Not exactly. Some Tibs 600 series are similar to those of SBST scanias, e.g.

    You are only half correct. You are correct on the point that TIB601Z to TIB999 is the same as SBS1Z to SBS399B, but it does NOT invalidate the rule that TIB(1000+X) is always the same as SBS(X). You are merely stating the dervied property that for any TIB number whose last 3 numerical digits are 600 and above, TIB(X) will have the same check digit as SBS(X-600).

    To demonstrate this fact, lets take an example stemming from the number 631 from SBS. If you go through the check digit derivation, you will find that the check digit for TIB1631 (1000 + SBS631) is E. The check digit for SBS1031 (TIB1631 - 600) is also E, but at the same time, back to itself, the check digit for SBS631 is also E !!

    Do you see what I mean?

    I know that TIB1631 does not exist yet, but you can go through the calculations yourself to see that the check digit is indeed E.

    In summary ->

    SBS631E, TIB1631E -> validates the property that TIB(1000+X) is the same as SBS(X)

    TIB1631E, SBS1031E - > validates the property that for any TIB number whose last 3 digits are 600 and above, TIB(X) is the same as SBS(X-600).

    Hence both properties are co-existent.

    The second mentioned property does not apply for any TIB number whose last 3 digits are 599 and below. You can do the calculations yourself to see what I mean. For example, since the last three digits of TIB1201 is below 600, TIB1201 (M) is not the same as SBS601 (S).
