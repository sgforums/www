    the AVMS DDU beeps at the instant it changes to +2 (Early by two minutes) and at -2 (Late by two minutes), with a different beep. Between -2 and +2 , the bus is "punctual".

    It may seem complicated, but I suppose they just insert the timings for the block for every stop and let the computer do the work.

    Once in a while the system server goes down and interchanges would show this:
    East AVMS DDU server down, manual update.

    There is also a number for the OCC which driver can call to if there is a problem when the system server is down.
