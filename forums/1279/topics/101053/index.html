    I guess it's got to do with the aprostophe in St Michael's Terminal..

    If SBS Transit/TransitLink is using Visual Basic Script Engine or some SQL/PHP/Perl script along that line, that's the problem.

    In some languages like C++ and Java, people use // to indicate comments. It's only for human to read, computer ignores everything after //.

    ////////////////////////////////////////////////////////
    // this is a line of comments in C++
    ////////////////////////////////////////////////////////

    Sometimes also can use /* */ pair to comment out stuff, what goes between is ignored by computer..

    int BusNum = Rand(999) /* Rand(999) calls up a number between 0 and 998 */ + 1;

    But for Visual Basic and VBScript,they use ' to indicate a line of comments.

    Dim Service as Integer ' Service is an integer value
    Dim OutofDepot as Boolean ' Either true or false

    So when it comes to St Michael's Terminal, unfortunately.. the data is stored at St Michael, the rest becomes comments, because this data is corrupted, cannot display properly (sometimes not at all!)

    Getaround method: use \', so St Michael's Ter becomes St Michael\'s Ter..
