    HTML form:



    Test Form












    Username:
    Email:





    The file that will get the email - submit.php

    submit.php contents:

    $connect = mysql_connect("localhost","root","");
    if (!$connect)
    {
    die('Could not connect: ' . mysql_error());
    }
    $name = $_POST['username'];
    $useremail = $_POST['email'];
    $to = "mayi@sgforums.com";
    $subject = "New user details";
    $message = "Here are the details" . "

    " . "Username is:". "$name" . "

    " . "

    " . "Email is:" . "$useremail" . "

    ";
    mail($to,$subject,$message);
    echo "Mail is sent."
    ?>

    Some explanation

    1. Usage of tables - that is to align it nicely. Try without tables it will look messy.

    2. Input type = "text" - text field, will have text box

    Input name = "name" - the name of the text box. This will later tell the submit.php file how to retrieve the details.

    3. Form action = "submit.php" - the file that will grab the details. They are all done at the server side, so the user can't glean details. With the exception of GET method.

    4. Form method = "post" - post method submit details without revealing in the browser.

    5. $name - just some variable name, can be string, can be integer

    6. $name = $_POST['username']; - if you used the POST method when creating the submission form, you need to use the $_POST method. Same thing for GET. It's $_GET. For a mixture, use $_REQUEST. The string in the square brackets is the name of the field in the HTML form.

    7. mail($to,$subject,$message); - it's a mail function in PHP.

    8. echo "string here" - like printing out the results in Java or whatever languages you know.

    9. $connect - connect to the database. In this format: database name, username, password. By default, most databases have no password.

    If you need help, refer to the documentation: http://www.php.net/manual/en/

    You can set up an Apache server, MySQL and PHP with XAMPP. You can consider XAMPP as a consolidation of the 3 open-sourced projects. Once installed, put all your files in this directory: \htdocs

    Start XAMPP, then type in localhost or 127.0.0.1 into your browser.

    Point to your file - for example, localhost/test.html and try submitting some forms.

    XAMPP by default loads all files from the htdocs directory. If you have other projects, create a new folder and store them there. Then point it to load this way: localhost/new project folder/test.html

    You can get XAMPP from here: http://www.apachefriends.org/en/index.html

    Originally posted by ndmmxiaomayi:
    For ASP and ASP.NET, install IIS. IIS can only be installed on Windows XP Pro. I think XP Media Centre also can. You need the XP installation CD to install. Not the Recovery CD.

    Start > Control Panel > Add/Remove Programs. On the left hand side, select Add/Remove Windows Components.

    Tick the Internet Information Services box and click Next. After that will be installing like any other programs.

    For PHP, you can use Apache. It works with IIS servers as well, but I don't know how to configure it.

    JSP... ask Java developers. Frankly I don't know Java.
    For Windows XP Pro with IIS, you can install PHP with the embedded ISAPI module. Just follow instructions here: http://www.php.net/manual/en/install.windows.php

    If you are using Windows XP Home, like me currently, you can install an Apache server on your computer before going back to the above link to learn about how to install PHP on Apache servers. To download Apache server, go here: http://httpd.apache.org/ Apache 2.x is recommended.

    For JSP, I ran a simple Google search on the internet. It seems like first you need to install the Java SDK on your comp first (http://java.sun.com/j2se/1.3/index.jsp). Then, following instructions here (http://www.stardeveloper.com/articles/display.html?article=2001060801&page=1), you need to install the Tomcat server before being able to deploy JSP it on your computer.

    For server-side scripts, I recommend a handy database as a nice addition. A good one would be the MySQL database Community version (it is free and available at http://dev.mysql.com/downloads/mysql/5.0.html). If I am not wrong, ASP used to be able to work with Microsoft Access. Not too sure about ASP.NET and JSP though.

    And lastly, before you can use ASP.NET, you need to install your .NET framework on your computer (available at http://www.microsoft.com/downloads/details.aspx?FamilyID=0856EACB-4362-4B0D-8EDD-AAB15C5E04F5&displaylang=en).

    Cheers.
