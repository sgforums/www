  • paulho
    13 Mar 04, 12:18
    But there is also another approach they have to take if really they want to change to a new tank ... seamlessly.

    Mi not a tankee but I think the key denominator here is the FCS, the FCS is the system or the software that currently all tankee crews are trained on it.

    eg: Makes no sense to install DOS 5 operating system currently used with your XT machine onto a pentium 4 machine because it's not efficient at all, better to upgrade the XT to start using Win2k instead, once you are familiar with win2k then you upgrade to a better machine.

    To change to a better tank assuming there is a need for it, first upgrade the FCS software on the amx to a superior one and train the crews to start using it.

    At the same time start phasing in or upgrade equipment for manufacturing of spares for new tank and retraining maintainence crews. Once your maintainence+spare support reaches critical mass level only then can you change the tank.

    Of course all this will also take a few years & cost a lot of money but in the end the rollout process will be quite a seamless cutover.

    just my thoughts.
