    I don't quite understand the issue with the source codes......

    If it's WVR weapons....you probably need to modify/add/control the HUD and HMD symbols? This one is problematic. I don't think you need source codes.

    If the mission computer is COTS then it's possible to plug (VME or Compact PCI) a RSAF specific processor card that can talk with the other existing cards. Probably at cost of greater time delay.

    Boeing can generate a set of APIs or Data and Control Packets for this specific processor to integrate the DASH or other weapons. But this require alot of planning to capture all the parameters you need for such integration.

    If it's BVR or standoff.....assuming it's WSO's job.....then it's possible to use another MMC with video multiplex/mix into existing display units. All the information to compute the firing solution should be on the 1553 databus.
