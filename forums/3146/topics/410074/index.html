[openBVE] Singapore Common Objects Dependency

  • joeyfjj
    4 Sep 10, 22:05

    The openBVE Singapore Common Objects Dependency (OsCod) is basically a object folder, in which there are all standard tracks, viaducts, tunnel, and other common miscellanous objects.

    This allows developers to refer to objects in this compiled objects, whilst allowing any updates to objects to be applied, and that update to immediately take effect in all routes utilising this. This will also reduce the file sizes of the routes, and these common objects will not be be duplicated.

    Website: http://joeyfoo.zxq.net/wordpress/bve-singapore/OsCod/

    Planned objects to be included:

    • "NSL" viaducts
    • "EWL East" viaducts
    • NSEWL tunnels
    • NSEWL tracks
    • NEL/CCL tunnels
    • NEL/CCL tracks
    • NSEWL signalling related objects
    • NEL/CCL signalling related objects
    • Passing train objects

    I would need developers to spend time building up these objects, as it is almost impossible to do it myself.

    Any comments, feedback, and such? Please post below!
