    openBVE development branch released - Better change ends thing! Look!

    "Some routes employ a fake "changing ends" concept where the user needs to jump the train to the next station from the menu which then looks like the previous station, just reversed. This continues to be the only way of simulating a change of travel direction, but from now on, openBVE support this concept at the coding level.


    Developers can now flag their stations for "changing ends", and the train will automatically jump to the next station with a gentle fading animation. Please see the Developing for openBVE pages, specifically the Track.Sta command for CSV routes. In short: Just use a C for the departure time instead of T."

    IN V1.6 of openBVE,

    Allow for routes to be loaded via external plugins. The API will be extended by the IRoute interface. The internal track geometry will be completely redesigned in order to allow networked tracks, but likely without functional switches or cross-rail signalling. At the end of this rather lengthy development process, routes can be built on which train services can run in any direction, including AI-controlled opposing traffic, an express train passing a local train, and the like. The CSV/RW formats will be greatly expanded so these features can be implemented in existing routes.
