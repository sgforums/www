    And where's an ATP plugin when you need it? Presenting the Singapore ATP plugin for openBVE, which I am creating (or rather, trying to create). 

    Developers wanting to reference the plugin should create the "ats.cfg" file in the train's folder. The content of the ats.cfg file should be "..\SgCommonFiles\SgATP\plugin.dll" without the quotes. 

    Users of such add-on shall need to download a compressed folder containing the plugin to extract to their Train directory. This way, the latest version of the plugin will always be used.
