    I got a question:

     

    I would like to ask a question on curves & converging / diverging tracks.

    Basically, I know that when a track is turning to the left, I can use .Curve -XXX where XXX is an integer and .Curve XXX for right curves. I also know the use of .Curve 0 = resume to straight line.

    I also do know that by specifying .rail 1;YY where YY is the distance away from .rail 0 (Main rail) in meters.

    But, I have a question on how to use them appropirately, i.e: What is the formula to specify these values so as NOT to create a extreme sharp cuve and causes discomfort of the passengers??

    Based on the Google Earth, the station KEM can be found on 1°19′15.54″N 103°54′46.23″E and station EUN can be found on 1°19′11.01″N 103°54′11.19″E and the code below will result a passenger discomfort during normal accleration when leaving station KEM and the curves between stations. By the way, Station KEM and EUN are the abbreviation of the station names in the real world.

     

    Sample code
    ; Last few statement on the first station, KEM
    1920,
    .stop 0;4;4,
    1925,
    .limit 80;-1;0
    ;Leaving KEM station towards the next station, EUN(50m straight as station itself, 50m slight left,75m sharp left)
    1975,
    ;Start of converging
    .curve 1000
    .rail 1;15.9
    2000,
    .rail 1;14
    2025,
    .curve 900
    .rail 1;11.3
    2050,
    .rail 1;8.6
    2075,
    .rail 1;5.6
    2100,
    .curve 0
    .rail 1;4
    ;Converged
    ;Turn right towards EUN for 175m
    2340
    .curve 1000
    ;Approaching EUN, give 85m of slanted straight track
    2515
    .curve 0
    ;Approaching EUN station ;Now diverge (75m slight left curve, 50m of left sharp curve, followed by 50m of minor curve)
    2600
    .curve -800
    .rail 1;4.2
    2625,
    .rail 1;5.5
    2650,
    .rail 1;6.8
    2675,
    .rail 1;8.1
    2700,
    .rail 1;10
    2725,
    .curve -500
    .rail 1;11.4
    2750,
    .rail 1;12.8
    2775,
    .curve -800
    .rail 1;14.6
    2800,
    .rail 1;15.5
    2825,
    .rail 1;17
    .curve 0
    ;Track has successfully diverged, 75m after here is the start of the station

     

    Note: The numerical value post in the code takes in the preceding station into consideration. Also, the commented section of how many meters for a curve etc are based from Google Earth and the ruler does not seemed to have a curve ruler (Or I just started to use it, so did not explore such feature??) which hence the amount of meters for a curve may not be absolutely correct.
