    First and foremost, you must learn the Java Programming Language syntax and semantics.

    Then when you want to have those graphical user interface, you need to consider buying *legal or pirated* Integrated Development Environment like JBuilder so you juz click, drag and drop those buttons, listboxes, combo-boxes etc onto the form. Easy like 123.

    If want to learn the hardcore way, use any text editor like Notepad, Wordpad and start keying in the code ala C, C++ style. Then you muz learn how to use/call those methods in the Java Abstract Window Toolkit or Java Swing Fountation Classes.

    Using those JBuilder etc tools require you to distribute the tools libraries files in addition to your own created classes file to the end user.

    To minimize dependency, do it the hardcore way and the benefit is that all the user has is to install the Java Runtime Environment and can run your program already. There are no additional JBuilder etc classes needed.

    Of course, you can try writing Java applets but you need a website to host it. So far I have problems in the website folder permission to turn my own Chuadaidi game into a Java applet. It runs fine in a stand-alone browser but failed when I tried to use URL.

    I did a spaceinvader-liked java game before while I was back in my poly days. Used AWT instead of swing cos I used Java 1.1 during that time, swing doesn't exist.

    I used threads to run the program. each class implements runnable and all are called by run() within the main program.

    I create classes for:

    -player's craft and data
    -enemy's craft and data
    -boss's craft(extends enemycraft) and data
    -secret craft (extends playercraft) and data
    -game controls, coors, level's control and event handlers (static)
    -background pic and coor
    -paint screen for double buffering and repainting. AWT needs to implement Doubling bufferingto reduce "flickering" but I think swing does it for you. unfortunately, I didn't manage to fix the flickers.
    -main menu GUI
    (all classes extends frame and are runnables except for The Menu GUI)

    I have no sound class cos Java 1.1 at that time only supports sound in applets and not application. I think now it does.

    hope these give you a rough idea on planning your objects creating a space invader game.
