    Okay, generally this problems occur when your system files which are critical to the kernel has been removed and thus its causes the kernel unstable due to absence of those files. Some of these files might be the driver files, which are necessary for the operating system for communication between hardware and software. In these instances, it is best to format your com and reinstall a new operating system(win98/win2k/winxp). But then again, second alternative will be reinstall the operating system over the existing one. This will recreate some of the crucial files that were deleted and replacing them. But this is not a good suggestion as your programs in return will be not that stable as some of the files that were linked to communicate between software and O/S are gone. Though your important documents are retained.

    File extensions will ".dll" are basically crucial files mostly being used by a few programmes. That means, one file to use for running multiple processes. Deletion of these files are normally by mistake. Cause normally the O/S will prompt user for confirmation to delete shared files and users more often than not, will click "yes" to delete them. These file size are generally small so its best to leaf it as it is. Only if you are sure the file is no longer needed, then you remove them. But sometimes its due to users remove program by force. Which means, delete the root folder from windows. This is superbly not recommended. Coz the O/S will not prompt user for confirmation to delete crucial files.

    To prevent O/S to screw up your crucial documents, it is best to partition your hard disk drive into 2 or more partitons. O/S on one drive, important documents on the other. Partition can be done using FDISK, which is present in most DOS and BOOT disk. But easiest method will be using a program called Partition Magic. It works well for all editions.
