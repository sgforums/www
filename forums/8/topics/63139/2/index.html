    Windows is a multi-tasking operation system, meaning it can run many programs simultaneously. The CPU, however, can only do one thing at a time. To solve this problem, Windows divides the CPU time into small slices and gives each running program a short period of time to execute, before the CPU is handed over to the next program. The more programs you run, the smaller each time slice gets. If the slices are too small, there may not be enough time for the burning software to fill the buffer on the CD writer and the CD writer will run out of data. Because the CD writer cannot wait, this will cause gaps or in the worst case, will ruin the entire disc.

    It is recommended that all the files you want to burn are located on your hard drive. Always cache to the hard drive before burning. (This is usually an option in your burning-software)

    Terminate all running programs except the CD burning software. This will maximize the CPU time slice for CD burning software. If possible, increase the priority of the CD burning software.(2k,XP)

    If still got problem, then
    Don't use your CD writer's maximum burnings speed. If your writer can burn at 48x, use 24x. This way, the CPU will only have to send half as much data to keep the buffer full, making the whole process less sensitive to errors.
