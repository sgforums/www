    the transaction is done by microprocessor link directly to the card reader. the software has to do the necessary decryption, validation, deduction/top up, encryption and so on within the split second. All these transactions will be kept until end of the day to be sent to the host for settlement. the host will tally all the transactions from singapore at night (usually). black listed card number will also be distributed during the settlement. this is how cash card works. ez-link should work in similar fashion, with more complex variants. at least the offline model is true. each real time / online transaction [e.g. NETS] through the host requires few seconds, so online transaction is not possible.
