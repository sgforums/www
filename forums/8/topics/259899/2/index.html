    As far as security is concerned, it looks all right to me. The only problem seems to the forum software, which actually cause all these problems.

    Of course, no forums is immune to a bot (the StookyUnited and whatever Seller clones spamming the forums at one time is caused by bots). One thing the admin could do is to tighten the registration process. That will hopefully prevent most automatic registration. One can also go one step further to make them read the User Agreement, embed some secret words there and tell them to fill in. Quite a few forums do this to prevent a bot. A bot can't figure out all these.

    There's another more extreme method, which is used by one of the forums overseas is to use your ISP email address, then a verification link is sent there. Not possible here, all Starhub users locked out. Not sure for PacNet...

    Then about the exploits (fonts turn ultra big after visiting a thread). Technically, the image tags are being read as shown by the red X. But it's not being read correctly.

    One way to stop all these nonsense will be to use PHP to check that there is a .jpg, .gif, .png (not case sensitive) extensions at the back of the image linked.
