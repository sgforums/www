    declare a link list struct first.

    struct link_list
    {
    int node_no;
    int item;
    int next_node;
    } list[4];

    Now what happen is that. Each cell in the list contains its node_no and the number of the next node.

    First node needs to store node_no of the second node, Second node stores the node_no of the third and so on and so on.

    So to link one node to the next, say node 5 to node 3, and node 3 to node 10, u do something like :

    list[5]->next_node = 3;
    list[3]->next_node = 10;

    Say, when node 3 gets deleted, u will have to unlink node 5 and node 10 from it. So u do something like

    list[5]->next_node = 10;

    With this step, u have plucked out node 3, and hooked up node 5 to node 10.

    Something .. something like that..

    The simplest implementation of a queue is through the use of array. Let's say we have a queue that is 5 item long

    int[5] = {1,2,3,4,5}

    So after popping the first item from the queue, we have

    {2,3,4,0}

    after popping the 2nd item,

    {3,4,0,0}

    Now let's say u wan to add 2 to the queue, it becomes

    {3,4,2,0}

    add 1 to the queue

    {3,4,2,1}

    Say, now 8 wants to join the queue. Can he do it? Well, that depends on your implementation.

    A queue is an example of a FIFO (or first in first out algo). Pretty straight forward because at no time can someone from the middle jump queue.
