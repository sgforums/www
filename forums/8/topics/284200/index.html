    If you can edit the Hosts file, maybe you can get it to load.

    Windows:

    Start > Run and type in notepad %windir%\system32\drivers\etc\HOSTS

    Add this line to the bottom, don't delete any other lines present:

    64.22.86.75 sgforums.com

    Linux:

    Open Bash aka Terminal and type in sudo (text editor name) /etc/hosts, press Enter.

    Enter the admin password (most likely your password) and press Enter. Note that there will be no asterisks shown when typing a password. Just proceed. Linux knows that password is being entered.

    Then add in this line to the bottom, don't delete any other lines present:

    64.22.86.75 sgforums.com

    If sudo doesn't work, use su.

    If sudo and su doesn't work, use sudo su.

    Nothing works, log out and log in as root.

    Username: root
    Password: Try your own password or guess your own distro default password.

    Note that for Ubuntu, Kubuntu, Xubuntu and their derivatives, there's no such thing as logging in as root. There's no root. Your only way is to use sudo.
