    [B]Or is there a simpler and more effective way to do it? Like using datagram or stuff like that? [/B]

    If you use just the J2SE SDK only this is the only way to do it. writeobject/writebyte will transfer an amount of data from the client to the server, but it wouldn't transfer a lot.

    So say if you have something like 2 meg to transfer from the server to the client, you have to transfer several times. TCP is good for this because a connection is maintained between client and server throughout the whole transfer process. So you simply just write the program to split 2 meg to around 4 packs of 500bytes. Then have the client request for them 4 times.

    At the end of the transfer, just join everything together. There is an issue here. Because 1) the packets might be lost on the way. 2)It is inefficient to transfer/request/transfer/request because half the time, the server will be sitting idle and waiting for the client to ask for something.

    The issue is resolved here. http://www.ce.chalmers.se/~fcela/javis/tcp_slidwin.html

    You can write the program to implement the sliding window algorithm. Actually no. Someone must have already written it elsewhere...if this isn't a school assignment u can search around the net for it.

    Or if you are allowed, use the HTTP Post library from the J2EE servlet library. That will simplify the problem altogether.

    How do I let an applet write a file?
    Applets loaded into a Java-enabled browser can't write files.

    Sun's appletviewer allows applets to write files that are named on the access control list for writing. The access control list for writing is empty by default.

    You can allow applets to write to your /tmp directory by setting the acl.write property in your ~/.hotjava/properties file:

    acl.write=/tmp

    You can allow applets to write to a particular file by naming it explicitly:
    acl.write=/home/me/somedir/somefile

    Use : to separate entries:
    acl.write=/tmp:/home/me/somedir/somefile

    Bear in mind that if you open up your file system for writing by applets, there is no way to limit the amount of disk space an applet might use.
