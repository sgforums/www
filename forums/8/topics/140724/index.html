    And Singapore currently has only ONE site in this top500 list. The accolade of having the fastest supercomputer system in Singapore belongs to Chartered Semiconductor Manufacturing (CSM) .

    The system they used is called Clustered Platform 4000 DL145 Opteron 2.2 Ghz Myrinet by Hewlett-Packard. It uses 384 AMD Opteron 2200 MHz processors all clustered and linked together using the Myrinet topology.

    The benchmark used to compile this top500 list is the LINPACK benchmark. The CSM system scored 1192.2 GFLops and is rank a mere #490 (ten place from bottom). I have been following this ranking since 2003 and this is probably the worst showing by Singapore. We use to have at least 3-5 sites in the top500. NUS, NTU, A*STAR and even Citibank Singapore use to be in the top500 list. Unfortunately they are now nowhere in the top500 list. That is how fast the supercomputer scene is changing and Singapore is apparently being left behind.

    give a thought to the rest of us who have to live without a supercomputer

    Of cuz with a faster supercomputer theoretically the applications are supposed to run faster. Hardware is one factor but not the determinating factor. Software also plays a part. If the software is written in a very inefficient way using lousy data structures and algorithm, no matter how good the hardware is, the intensive load pumped into the software will also slow the overall performance. It is always so easy to say slow then get faster hardware lor. Before one attempts to take that approach, one gotta check if the software is indeed the culprit or not.

    Through my working experience, very often it is the combination of software and hardware that contribute to 'slowness'. Most of the time, efficient software do can compensate for the hardware slowness. One major factor is I/O. Software that always write and read from database is going to suffer. Some jobs should be processed in memory and the final output written to disk. That is the ideal approach.
