    Start -> Run -> "Cmd"
    In the console, type "ping "

    Usage: ping [-t] [-a] [-n count] [-l size] [-f] [-i TTL] [-v TOS]
    [-r count] [-s count] [[-j host-list] | [-k host-list]]
    [-w timeout] target_name

    Options:
    -t Ping the specified host until stopped.
    To see statistics and continue - type Control-Break;
    To stop - type Control-C.
    -a Resolve addresses to hostnames.
    -n count Number of echo requests to send.
    -l size Send buffer size.
    -f Set Don't Fragment flag in packet.
    -i TTL Time To Live.
    -v TOS Type Of Service.
    -r count Record route for count hops.
    -s count Timestamp for count hops.
    -j host-list Loose source route along host-list.
    -k host-list Strict source route along host-list.
    -w timeout Timeout in milliseconds to wait for each reply.

  • Icemoon
    22 Apr 06, 22:11
    Originally posted by StarPuppy:
    ya but for what main purpose?

    For the user to check there is a path from your IP to the target IP, and how 'laggy' is the data transfer.

    It is very useful when you're setting up internal network. you'd want to make sure the two computers (or your computer to a router) are connected before you try things like file and printer sharing.

    I always use it when my browser suddenly takes very long to load something, like google. I ping google, if no response means no connectivity. If this persists, need to go restart the modem.

    It can be used as an objective measure. For example, you want to boast to your friend your connection has less lag. So both of you do a ping. The one who has lesser time (in ms) wins.
