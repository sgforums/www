    The new system run on Microsoft platform and the e-filing web application was brought down numerous times on the 30th April. It displayed ugly error messages [database error messages] which any web designers know by instinct should be hidden from end users in case of system failure or application exceptions. At times, it simply redirect the end users back to the main page [mytax.iras.gov.sg] after the Singpass login without any messages. For certain instances, I noticed that some debug messages will be displayed at the top left hand corner of the main page. One example is a small capital letter 'f', at other times it was a string of other technical debug messages, useful only to programmers.

    Besides, if you have just registered your Singpass at the CPF counter, I encountered [by helping friends] that they were able to log in using the new Singpass into CPF e-service but not the IRAS portal. Whereas I am able to do with my Singpass which was registered many years ago. The login was simply redirected back to the main page without any messages, or at times, as the system likes, some machine-readable-only messages again.

    I believe that IRAS aims to provide a better service. However, I think the choice of system platform, web application design, exception handling has much to improve.
