    An IDE (Integrated Drive Electronics) interface is a standard way for a storage device to connect to a computer. IDE is actually not the true technical name for the interface standard. The original name, AT Attachment (ATA), signified that the interface was initially developed for the IBM AT computer.

    IDE was created as a way to standardize the use of hard drives in computers. The basic concept behind IDE is that the hard drive and the controller should be combined. The controller is a small circuit board with chips that provide guidance as to exactly how the hard drive stores and accesses data. Most controllers also include some memory that acts as a buffer to enhance hard drive performance.

    Serial ATA (SATA) and Parallel ATA (PATA) are both IDE specifications, but commercially most people refer to PATA drives as the generic IDE drives.

    SATA technology promises faster interface speed at 150 MBps in theory, but doesn't really improve performance as compared to the older PATA speeds at 100/133 MBps. The revolution speed of your physical hard drives matters more.

    The good things about SATA drives are that the cables are thinner (which could improve air flow within the hardware system), and that they don't require jumper settings (thanks to Hot-Swap support). The bad thing is the stability issue, which is debatable though. However, there is still much development potential in the SATA technology, so it's probably the way to go.
