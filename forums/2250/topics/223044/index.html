    Number 1: Badongo has prided itself as Hacker Safe. This means they most likely own server(s) themselves, in order to have the maximum control over security. You don't want files to disappear or appear at will.

    Number 2: 100GB for a website like Badongo, you probably have to clear files every 3 days if millions are going to use it. Not a good idea, because other hosts make sure it's at least 30 days.

    Number 3: I suggest you read more about SEO - Search Engine Optimization. If you know nothing about SEO and want to be as successful as Badongo, then dream on. For a start, have a read at Geeks to Go to find out what's SEO is all about.

    Number 4: Did you even plan? What sort of database are you going to use? If your website is going to have files uploaded, they have to be stored somewhere, not anywhere. A suitable language for querying the database is also needed. HTML can't do everything. Badongo is not just set up in a night, it takes lots of preparation work, discussing and designing.

    Number 5: Did you test for compatibility issues for the different browsers available? IE is not the only browser that users use, mind you. If your website is going to show up funnily to users using other browsers, they will be pissed off. One way to resolve this issue is to write a script that detects the type of browsers used. They are readily available off the web, take one and copy.

    Number 6: How are you going to deal with users who disable scripting on their browsers? How do you detect this?

    Think carefully through all these points. If you already have a website done up, have your scripts written, then find a free web host to test it out before paying for your real web hosting.
