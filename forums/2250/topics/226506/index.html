    Either you attack via server or you attack via the user.

    Attacking via user would be easiest.

    Another way of attacking would be called the "man in the middle" attacks.

    Like its name, you are right in the middle, between Blogger server and the user. You will have to figure a way to intercept the user's packets before it gets to the server, and route it to the Blogger's server as if nothing has happened.

    This seems easy, but it isn't. Administrators are always on the lookout for such attacks, because they will cause a particularly spike in traffic graphs. And if the user has good technical knowledge, this can be discovered as well.

    Man in the middle attacks, if not handled carefully, could land you in jail. Don't think that Singapore's ISPs use proxies to hide your IP addresses, hell no.

    Attacking via user would be some sort of phishing. You will have to domain squat, that means buy a domain that spells exactly the same as blogger or spells similarly to blogger.

    Attacking the server is not easy if you have no technical know-how and no knowledge of other types of operating systems except Windows.

    Should be enough for you to figure out.
