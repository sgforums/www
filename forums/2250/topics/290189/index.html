    Originally posted by LatecomerX:
    Currently I have DHCP enabled on the router, so is it possible that it will assign another client the same IP address as mine when it is connecting to a DHCP-enabled computer since mine is not in the DHCP list, resulting in some clashing there? Or will DHCP prevent that?
    The solution is to use a static IP outside of the DHCP range. For example, if your DHCP server is configured to issue 192.168.1.100-192.168.1.253, use 192.168.1.99 as your static IP.

    Let's say if you have a static IP that's within the DHCP range. If you are already plugged into the network, the DHCP server will not issue your static IP as it already exists on the network. However, say you turn off your PC and another PC joins the network and obtains that IP. Once you rejoin the network with your static IP, there will be an IP address conflict.

    However, it's best to avoid this issue completely by using an address outside the DHCP range.


    Will there be issues if I leave both DHCP enabled on the router and the computer while using a static IP address to connect to the router? Wouldn't there be some sort of race condition to assign an IP address to the computer? Or does DHCP prevent this from happening as well?


    No. If you have a static IP, your laptop will not send a DHCP request. There should be no problem leaving the DHCP server enabled on the router.

    You MUST have DNS set in your laptop. if you do not, your laptop will not be able to resolve domain names. It's up to you whether you want to enable the DNS relay server in your router. This usually just acts as a DNS proxy for your ISP DNS server (unless you have a really advanced router). Personally, I prefer to manually configure the ISP DNS like he has done.

    If you think you have problems with your DNS, try flushing the DNS cache on your laptop.

    ipconfig /flushdns
