    If you are using recovery disk, you may not require to enter your serial number again.

    1) When there is no hard disk activity doesn't mean your pc halted. Please tell us the time you waited before you deemed is as hanged (you may have to wait up to 15~30 mins, please be patience).
    By the way, you insert you CD and it straight go into Microsoft splash screen?

    2) Why are you formatting your pc at the first place? Windows setup hang might have something to do with your format problem.

    3) There isn't any memory allocated in your system yet, except DOS (since I assume you hasn't installed any OS yet). Therefore Windows Memory Diagostic test should have no result.

    4) If you disable those devices and can install, after you finished install Windows, and you have to use them, it hang again how?

    Maybe I can suggest you to enter BIOS and set halt on "no error".

    Thanks for the tips guys, I have the problem resolved. Apparently setup was unable to proceed further because of the processor problem. Here's the KB that resolved my problem (Take note that I did not flash my BIOS to resolve this issue, and I'm formatting a DIY PC):

    http://support.microsoft.com/kb/885626/en-us

    Here's the link that will help you identify those processors that will cause conflicts with WinXP:

    http://support.intel.com/support/processors/tools/frequencyid/sb/CS-007616.htm

    Taken from MS KB:

    Computers that are affected by this issue have the following specifications:
    - CPU Family value of F.
    - CPU Model value of 3.
    - CPU Stepping value of 3.
    - CPU Revision value of 3, 2, 1, or 0.
