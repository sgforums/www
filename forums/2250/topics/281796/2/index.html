    accesor methods are used to obtain the values of object states. Another term used is getter method

    Mutator methods are used to set/initalize values of object states.Another term used is setter method

    Function Overloading.One of the beauties of the object oriented paradigm.

    gived this 2 method signature

    performCalculation(int days,double basic salary,String position)
    performCalculation(int days,double basic salary)

    They have the same method name,except they differ in method signature. These are used as means by a programmer to expose interfaces of different input parameters to callers on a basic level due to different business requirements.In computer science speak, its called Polymorphism.

    There is another form of polymorphism called method overiding.This is usually perform at the inherited class level. 1 good example will be the toString() method in the J2SE API.

    By default when you type this.toString(); information abt the instantiated object is displayed to the standard output.usually in a derived class, you will overwrite the method by declaring the same method name & signature. But within the method body, you will write your own code to display what is deem fit for your object as determined by the functional specs determined by your application architect/designer.
